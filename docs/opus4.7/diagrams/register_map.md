# voxel_gpu register map

**Source of truth.** Word addresses are localparams in
`hw/voxel_gpu/rtl/voxel_gpu.sv` (`ADDR_*`). Byte offsets and bit
mnemonics are macros in `sw/voxel_gpu.h`. The Avalon-MM lightweight
slave is **word-addressed** (`addressUnits = WORDS` in
`hw/voxel_gpu_hw.tcl`), so the C-side byte offset is `4 * word_addr`.

The map below was generated by hand against those two files; the
script `scripts/gen_pipeline_diagram.py` also emits the same merged
table into `build/diagrams/voxel_gpu.json` (`register_map` key).

## Control + status

| Byte off | Word | RTL name (`voxel_gpu.sv`) | SW macro (`voxel_gpu.h`) | Dir | Notes |
|---:|---:|---|---|:--:|---|
| 0x0000 | 0x000 | `ADDR_CONTROL` | `VOXEL_REG_CONTROL` | R/W | `[0]=EN`, `[1]=FLP`, `[2]=IEN`, `[3]=CLR` |
| 0x0004 | 0x001 | `ADDR_STATUS` | `VOXEL_REG_STATUS` | R | `[0]=BSY`, `[1]=FFL`, `[2]=FEM`, `[3]=VSY`, `[19:4]=FIFO_COUNT` |
| 0x0008 | 0x002 | `ADDR_FRAMECNT` | `VOXEL_REG_FRAME_COUNT` | R | free-running frame counter |

## Band controls

The 60-line band is the unit of rendering. `gpu_transport.c` writes one
band at a time: BAND_INDEX → FIFO descriptors → `BAND_CTRL.BEGIN` →
descriptors stream → `BAND_CTRL.FLUSH`.

| Byte off | Word | RTL name | SW macro | Dir | Notes |
|---:|---:|---|---|:--:|---|
| 0x0034 | 0x00D | `ADDR_BAND_INDEX` | `VOXEL_REG_BAND_INDEX` | R/W | 0..`VOXEL_BAND_COUNT-1` (= 7) |
| 0x0038 | 0x00E | `ADDR_BAND_CTRL` | `VOXEL_REG_BAND_CTRL` | W | `[0]=BEGIN`, `[1]=FLUSH` (pulses) |
| 0x003C | 0x00F | `ADDR_BAND_WINDOW` | `VOXEL_REG_BAND_WINDOW` | R/W | `[5:0]=flush y0`, `[13:8]=flush y1` |

## Bit mnemonics (sw/voxel_gpu.h)

```
VOXEL_CTRL_EN            (1u << 0)
VOXEL_CTRL_FLP           (1u << 1)
VOXEL_CTRL_IEN           (1u << 2)
VOXEL_CTRL_CLR           (1u << 3)

VOXEL_BAND_CTRL_BEGIN    (1u << 0)
VOXEL_BAND_CTRL_FLUSH    (1u << 1)

VOXEL_STAT_BSY           (1u << 0)
VOXEL_STAT_FFL           (1u << 1)
VOXEL_STAT_FEM           (1u << 2)
VOXEL_STAT_VSY           (1u << 3)
VOXEL_STAT_FIFO_SHIFT    4
VOXEL_STAT_FIFO_MASK     0xFFFFu

VOXEL_EXTMEM_CTRL_ENABLE             (1u << 0)
VOXEL_EXTMEM_CTRL_SCANOUT_EN         (1u << 1)
VOXEL_EXTMEM_CTRL_BACKBUF_EN         (1u << 2)
VOXEL_EXTMEM_CTRL_RGB565             (1u << 3)
VOXEL_EXTMEM_CTRL_TILE_CACHE_EN      (1u << 4)
VOXEL_EXTMEM_CTRL_SKY_GRADIENT_CLEAR (1u << 5)
```

## ioctl ABI

`VOXEL_IOC_MAGIC = 'v'`. Numbers below are encoded by the
`_IO`/`_IOR`/`_IOW` macros in `sw/voxel_gpu.h`.

| ioctl | Direction | Payload | Effect |
|---|:--:|---|---|
| `VOXEL_IOC_CLEAR_FRAME` (1) | — | — | Writes `CLR` into CONTROL |
| `VOXEL_IOC_FLIP` (2) | — | — | Writes `FLP` into CONTROL, polls VSY |
| `VOXEL_IOC_SET_PALETTE` (3) | W | `voxel_palette_entry` | PAL_ADDR + PAL_DATA |
| `VOXEL_IOC_GET_STATUS` (4) | R | `voxel_status` | Reads STATUS |
| `VOXEL_IOC_GET_FRAME_COUNT` (5) | R | `__u32` | Reads FRAME_COUNT |
| `VOXEL_IOC_SET_FOG` (6) | W | `voxel_fog_state` | FOG_RANGE + FOG_CTRL |
| `VOXEL_IOC_SET_EXTMEM` (7) | W | `voxel_extmem_state` | EXTMEM_* writes |
| `VOXEL_IOC_GET_EXTMEM` (8) | R | `voxel_extmem_state` | EXTMEM_* reads |
| `VOXEL_IOC_BEGIN_BAND` (9) | W | `voxel_band_state` | BAND_INDEX + BAND_WINDOW + BAND_CTRL.BEGIN |
| `VOXEL_IOC_END_BAND` (10) | — | — | BAND_CTRL.FLUSH + poll BSY |
| `VOXEL_IOC_FLIP_ASYNC` (11) | — | — | Writes FLP without waiting |
| `VOXEL_IOC_WAIT_FLIP` (12) | — | — | Polls VSY |
| `VOXEL_IOC_GET_PERF` (13) | R | `voxel_perf_counters` | PERF_DRAW_*, PERF_FLUSH_*, PERF_INIT/LOAD |
| `VOXEL_IOC_GET_PERF2` (14) | R | `voxel_perf_counters_v2` | adds PERF_FLUSH_WAIT_* fields |
| `VOXEL_IOC_SET_SKY_PALETTE` (15) | W | `voxel_sky_palette_entry` | SKY_PAL_ADDR + SKY_PAL_DATA |

## Descriptor layout (FIFO_WINDOW payload)

64 B base descriptor (`struct quad_desc`) optionally followed by a 36 B
UV extension (`struct quad_desc_uv`) when `QUAD_FLAG_TEX` is set in
`flags`. Total descriptor length is therefore 64 B (color quads) or
100 B (textured quads). All fields are little-endian; see
`sw/voxel_gpu.h` for the exact struct definitions.
